## 一、诊断结论

### 问题1: Crypto数据采集中返回空 `{}`
**根因: OKX API因GFW SSL握手失败（443端口被阻断），永远返回空/错误**

- 测试确认: `Invoke-RestMethod 'https://www.okx.com/api/v5/market/ticker?instId=BTC-USDT'` → "无法连接到远程服务器"
- OKX域名在大陆被GFW阻断443端口，SSL/TLS握手阶段即失败
- `collect-prices-simple.ps1` (v7) 把OKX放在CryptoCompare之前作为主源，每次都先尝试OKX然后才到CryptoCompare
- OKX失败后3秒超时才降级，但CryptoCompare实际上可用（测得72727 USD）
- **修复**: 将CryptoCompare移到第一位作为主源，OKX完全移除，Gate.io作为备用

### 问题2: HN数据8天未更新（最后更新 2026-04-09）
**根因: HackerNews采集逻辑不完整**

- `collect-tech-news.ps1` 主要采集AI博客，HN只作为补充数据
- 采集脚本调用链中HN采集步骤未被正确调度或依赖项缺失
- HN (news.ycombinator.com) 在本环境无法访问（PowerShell直接请求超时）
- **结论**: HN在中国大陆被GFW屏蔽，必须通过Bing搜索代理或其他方式访问
- **建议**: 如需HN数据，需通过Bing搜索HN相关话题作为替代方案

### 问题3: macro-playwright脚本的黄金/原油/F&G丢失
**根因: IE COM无法获取JS渲染内容 + 合并逻辑缺陷**

- `collect-macro-playwright.ps1` 使用IE COM访问 goldprice.org / oilprice.com / cboe.com
- 这些网站全部是JS渲染，内容在初始HTML中不存在，IE COM抓不到
- IE COM抓取失败后：`$result` 中没有GOLD/OIL/VIX键
- **但关键BUG**: 旧版合并逻辑会执行 `macro = @{}`（空哈希表），即使IE COM没抓到任何数据
- 后续读取 prices_latest.json 合并时：`$finalData.macro` 是空的，会覆盖掉之前采集的所有macro数据！
- **修复**: 重写合并逻辑，只有当本次成功采集到数据时才更新 prices_latest.json；如果本次无有效数据则跳过更新（保留已有数据）
