- **根本原因**：Git全局配置存在错误代理

- Windows系统代理（netsh）正常：Direct access (no proxy server)

### 修复操作
`powershell
git config --global --unset http.proxy
git config --global --unset https.proxy
`

推送成功：c4be0fa..72a16cc  main -> main

之前配置的代理 127.0.0.1:7890 已失效（代理服务未运行），导致所有Git请求被重定向到不存在的本地代理。
