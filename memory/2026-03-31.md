## 07:01 宏观数据采集Cron集成子智能体完成

### 任务结果
**目标**: 将宏观数据采集(`collect-macro-playwright.ps1`)集成到Cron工作流

### 执行过程
1. **检查Cron任务**: Windows Task Scheduler中有`hourly-briefing`任务，每45分钟触发一次
   - 实际由`auto-push.ps1`驱动，调用`collect-prices-simple.ps1`
   - `collect-prices-simple.ps1`已有GOLD/OIL/VIX采集逻辑（但依赖JS渲染网站，IE COM失败率高）

2. **找到root cause**: `collect-prices-simple.ps1`中的SSL证书绕过代码破坏了OKX/Gate/CryptoCompare API调用
   - `[System.Net.ServicePointManager]::ServerCertificateValidationCallback = { $true }` 导致HTTPS连接关闭
   - 修复：移除SSL bypass，改用仅强制TLS1.2（`[Net.SecurityProtocolType]::Tls12`）
   - **同时**：删除了整个`try/finally SSL`外层结构，简化代码

3. **集成方案**: 采用**选项A**（修改现有`collect-prices-simple.ps1`）
   - 当GOLD/OIL/VIX初始采集失败时，调用`collect-macro-playwright.ps1`作为降级
   - `collect-macro-playwright.ps1`采用IE COM浏览器方式（CBOE.com的VIX有效，goldprice.org/oilprice.com被Cloudflare拦截）
   - 合并逻辑：先写crypto中间结果→运行浏览器降级→合并macro到prices_latest.json

4. **修复其他bug**: Substring空值崩溃（`$r.content.Length=0`时）
   - 所有`$r.content.Substring(0, ...)`改为`if ($r.len -gt 0) { $r.content.Substring(0, ...) } else { "" }`
