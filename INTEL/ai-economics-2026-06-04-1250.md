# AI 经济学情报:企业级 AI 治理的"经济线"与"技术线"

## BLUF (Bottom Line Up Front)

- **Uber $1500/月/工具限额**:首次出现大厂明确披露的"per-engineer AI 成本基准"——~$36K/工程师/年,占中位薪资包 11%,标志企业 AI 支出从"无序扩张"转入"硬性封顶"。
- **Anthropic Containment 框架**:首次系统披露 Claude 在 claude.ai/Code/Cowork 三大产品中的三层防御体系(环境沙箱+模型层+外部内容),并承认 Mythos Preview 因 blast radius 过高被拒发。
- **关联信号**:经济线(成本封顶)与技术线(风险封顶)在 2026 年同步成熟,企业级 AI 部署从 PoC 狂热转向**可治理采用**阶段,Anthropic 同时是 Uber 限额政策的"目标供应商"。

## Uber 限额核心

Uber 为每位员工设定了 **$1,500/月/AI 编码工具** 的 token 支出上限,仅适用于 agentic coding 工具(如 Cursor、Claude Code),各工具额度独立不互通。Bloomberg 报道确认该政策已在近月实施,源于 Uber 在 2026 年前四个月即耗尽全年 AI 预算——主因 2025 年制定预算时无法预测 token-burning 编码代理的爆发。Simon Willison 测算:按每位工程师同时使用 2 个工具,年化上限为 $3,000×12 = **$36,000**,约相当于 Uber 美国软件工程师 $330K 中位薪资包的 **11%**。该限额被作者评价为"对超支的理性政策回应",远优于"tokenmaxxing"排行榜式激励。个人开发者通过 Anthropic/OpenAI 补贴计划仅付 $100/月即可使用 ~$1,000/月 token 量,印证企业失去补贴后成本陡增。

## Anthropic Containment 核心

Anthropic 首次系统披露三大产品(claude.ai、Claude Code、Cowork)的**三层防御+三种隔离模式**架构。核心框架:风险分三类(User misuse / Model misbehavior / External attackers),防御覆盖环境层(沙箱、VM、egress)+ 模型层(系统提示、分类器)+ 外部内容层(MCP、第三方插件)。关键数据:**Claude Code 用户同意 93% 的权限请求**,导致"approval fatigue"——这是 human-in-the-loop 失败的核心证据,催生了自动批准低风险操作的 "auto mode"(捕获 83% 过度行为)。在 Gray Swan 基准上,Claude Opus 4.7 单次 prompt injection 攻击成功率 0.1%,100 次自适应攻击升至 5-6%。三种隔离模式:claude.ai 用 gVisor 临时容器(无持久工作区),Claude Code 用人机协作沙箱(参考 devcontainer 允许无人值守),Cowork 提供更深度文件系统访问。文章承认 **Mythos Preview 因 blast radius 过高未在 2026 年 4 月发布**,反映"能力扩张 > 安全防御"的核心张力。

## 关联分析

两条信号分别从**经济**与**技术**维度,标志 2026 年成为**企业级 AI 治理元年**。Uber 的 $36K/工程师/年首次给出"AI 工具价值锚点",为行业 HR/财务提供预算参考;Anthropic 的 containment 框架则承认"无限扩张能力"不可持续,必须用沙箱封顶 blast radius。**关键耦合点**:Anthropic 既是 Uber 限额政策的目标供应商,又通过 auto mode 主动降低单次操作成本——这意味着 Claude Code 用户的实际"成本/价值比"会因 auto mode 进一步下降,但**单次操作的潜在破坏半径**反而上升。两端共同推动市场从"AI 工具军备竞赛"过渡到"AI 工具精算时代"。
