## 2. ALERTS清理方案

### 2.1 分类算法
```
输入: ALERTS/*.md文件
├─ 噪声关键词匹配 → NOISE (96.5%)
│  ├─ 关键词: okx_btc, binance_btc, API健康, health FAIL, score=40%, gfw_health FAIL
│  └─ >14天前 → ARCHIVE\alerts\
│  └─ ≤14天 → 保留(后续自动过期)
├─ 信号关键词匹配 → SIGNAL (1.3%)
│  ├─ 关键词: F&G, BTC暴跌, FOMC, CPI, NFP, 地缘, Hormuz, 石油, CRITICAL, SECURITY
│  └─ 永远留在ALERTS/
└─ 无匹配 → UNCLASSIFIED (2.2%)
    └─ 人工审核后补充关键词
```

### 2.2 归档策略
- **阈值**: >14天的噪声文件 → `ARCHIVE\alerts\`
- **预计归档**: ~700个文件(截至6/9之前的全部噪声)
- **保留原位**: 10个信号 + 17个未分类(待审) + 近期噪声(约80个)
- **清理后ALERTS预期**: ~107个文件，可大幅降低

### 2.3 脚本: `scripts\alert-cleaner-v1.ps1`
```powershell
.\scripts\alert-cleaner-v1.ps1
```
功能：
- 智能分类 (关键词匹配)
