# 200字事件核心
Gooey（chriskiehl/Gooey）是 Python 生态里最长寿的"CLI→GUI"转换工具。核心机制是 `@Gooey` 装饰器：开发者照常用 argparse 写命令行参数，装饰器自动解析出每个参数的类型（int/str/FileChooser/DateChooser...），渲染出对应表单控件，运行时通过子进程把用户输入回灌到原 CLI。架构风格类似 argparse-s原来该领域的 de facto 标配。社区反响：HN 评论区反复出现"If you build it as a CLI, you can get the GUI for free via Gooey"——已成圈内 meme。最近 2026-03-23 仍有提交，活跃度尚可。**注意**：任务原始 URL `answerdotai/Gooey` 返回 404（GitHub API 确认 Not Found），实际项目归属 `chriskiehl/Gooey`；主代理派单时已贴错链接。

# 150字技术解构
- **架构**：装饰器模式（decorator-based），反射 argparse 的 ArgumentParser 自动生成 UI；可选 GooeyParser 子类做精细控件控制（Dropdown、MultiFileChooser 等）
- **依赖栈**：纯 Python，底层 wxPython（跨平台原生控件，非 Electron/Qt）；运行时把 GUI 进程作为父进程、子进程跑原 CLI，stdout/stderr 回流到日志面板；支持 i18n JSON 国际化
- **与同类对比**：vs PySimpleGUI（更通用但需手写布局）vs Textual（TUI 而非 GUI）vs Streamlit（Web 而非桌面）；Gooey 的杀手锏是**零代码改动**——你已有的 argparse CLI 加上 `@Gooey` 立刻得到 GUI，对老项目零侵入
